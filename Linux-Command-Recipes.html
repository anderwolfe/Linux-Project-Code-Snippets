Linux Command Recipes — Beginner Edition

Simple, linear recipes for building commands step by step.

Symbol Reference

~ → home directory

/ → root directory

* → wildcard (match anything)

. → current directory

.. → up one level

Understanding the Prompt

The command prompt usually shows who you are and where you are in the filesystem.

Example Prompt
student@machine:/etc$
  │                └─ current directory (/etc)
  └─ user (student)

This helps you know what “here” means when using commands like ls or cd.

cd — Change Directory

Purpose
Move between folders in the filesystem.
Steps
1) Base Tool
cd
2) Target
Path of the folder you want to enter (.. up one, ~ home)
Examples
Go to your home directory
cd ~
   └─ tilde (~) means home
Go to the /etc directory
cd /etc
     └─ absolute path from root

pwd — Print Working Directory

Purpose
Show the full path of your current folder.
Steps
1) Base Tool
pwd
Examples
Where am I?
pwd
 └─ prints your current directory path

ls — List Directory

Purpose
View directory contents with options for details, hidden files, or sorting.
Steps
1) Base Tool
ls
2) Flags: Options
-l details, -a hidden, -h human sizes, -t sort by time
3) Target
What location would you like to list (this is optional, if left blank it will list “here”)
Examples
Details + hidden
ls -la
     └─ -l details, -a all files includes hidden files
Newest files first
ls -lt /etc
     │   └─ what location you would like to list
     └─ -l details, -t sort by time

mkdir — Make Directory

Purpose
Create new folders. Use -p to create parent folders as needed.
Steps
1) Base Tool
mkdir
2) Flags
-p use this flag if you want to make folders within folders with this single command
3) Target
What would you like to name your folder. You can also put a location before the name to make the folder somewhere other than here. (e.g. usr/place/foldername instead of just foldername)
Examples
Create a single folder
mkdir notes
        └─ folder name to create
Create nested folders if needed
mkdir -p logs/2025/09
       └─ -p creates folders inside of folders if necessary

rm — Remove

Purpose
Delete files or directories. ⚠️ Destructive — use carefully.
Steps
1) Base Tool
rm
2) Flags
-r this flag is necessary to remove folders, -f force (tells computer, do this command no matter what), -i interactive (tells computer, ask me questions while you're deleting things)
3) Targets
Name of the file or folder you would like to remove
Examples
Delete one file
rm old.txt
     └─ file to remove
Delete a folder and everything inside
rm -r backup/
    └─ -r removes your target, but also anything inside of it
Force delete many logs
rm -f *.log
    │  └─ wildcard: all files ending in .log
    └─ force (tells computer: don't ask any questions, just do it)

mv — Move or Rename

Purpose
Move files/folders to a new location or rename them.
Steps
1) Base Tool
mv
2) Source(s)
Old file or folder you would like to move/rename
3) Destination
New name or folder path
Examples
Rename a file
mv draft.txt final.txt
      │          └─ new name
      └─ old name
Move a file into a folder
mv report.pdf /tmp/
     │           └─ where to put it
     └─ what to move

cat — Show File Contents

Purpose
View a file's contents in the terminal, or join multiple files together.
Steps
1) Base Tool
cat
2) Targets
Name of file you would like to view
3) Redirection (optional)
Use > to write output into a new file
Examples
View a file
cat notes.txt
      └─ file to display
Concatenate two files into a new one
cat a.txt b.txt > both.txt
       │    │          └─ new file
       │    └─ second file (added after first)
       └─ first file

grep — Search Text

Purpose
Search for text inside of a file.
Steps
1) Base Tool
grep
2) Pattern
Quote the text to search for, e.g. "error"
3) Targets
What file or folder do you want to search?
4) Flags: Modifiers
-i says don't worry about uppercase and lowercase, -n show line numbers, -R if there's folders inside of what we're searching, search those too
Examples
Search for the word "error" in a file
grep "error" app.log
        │       └─ search inside of this file
        └─ search for: "error"
Search for "error", ignoring case and showing line numbers
grep -in "error" app.log
       │     │      └─ search inside of this file
       │     └─ search for: "error"
       └─ -i ignore case, -n line numbers
Search recursively for "main()" in source folder
grep -R "main()" src/
      │     │       └─ search inside of this folder
      │     └─ search for: "main()"
      └─ -R recursive (meaning if there's folders inside of what we're searching, search those too)

find — Locate Files

Purpose
Search for files by name, size, type, or modified time.
Steps
1) Base Tool
find
2) Path
Where to search (e.g. . for here)
3) Flags: Type
-type f only search for files, -type d only search for folders (also known as directories)
4) Flags: Filters
-name name we're searching for, -size by size, -mtime by modified time
Examples
Find "logfile" in current folder
find . -type f -name "logfile"
      │       │          └─ filename to search for
      │       └─ -type f (files)
      └─ where to search (. means here)
Find files larger than 50 MB in /var
find /var -type f -size +50M
       │     │        └─ size filter
       │     └─ -type f (files)
       └─ where to search (/var)

chmod — Permissions (Symbolic)

Purpose
Set permissions for user/group/others using symbolic form.
Steps
1) Base Tool
chmod
2) Who
u = user (owner), g = group, o = others, a = all
3) Operators
= set exactly, + add, - remove
4) What
r read, w write, x execute
5) Target
Which file or folder would we like to change the permissions of
Examples
User gets read & write
chmod u=rw file.txt
        │      └─ change the permissions of this file
        └─ who= u (user), what = rw (read and write)
Group & others read-only
chmod go=r notes.txt
        │     └─ change the permissions of this file
        └─ who = g (group), o (owner)  what = r (read)
User read, write & execute
chmod u=rwx script.sh
        │       └─ change the permissions of this file
        └─ who = u (user) what = rwx (read, write, and execute)

tar — Archives

Purpose
Create, extract, or list compressed archives.
Steps
1) Base Tool
tar
2) Flags: Main Action
-c create, -x extract, -t list
3) Flags: Compression
-z gzip, -j bzip2, -J xz
4) Flags: File Handling
-f always include f flag when using tar
5) Targets
What is the name of the archive we are working with (include file extension in this name, so like .tar at the end of the name)
Examples
Create a gzip archive with a folder inside
tar -czf backup.tar mydir/
      │     │          └─ what to put inside your archive
      │     └─ what you want to name your tar archive (make sure to include .tar at the end of name)
      └─ -c create, -z gzip, -f (always do f)
      
Extract from an archive
tar -xf backup.tar
      │   └─ name of the archive you want to extract from
      └─ -x extract, -f (always do f)

      
Look inside an archive (list)
tar -tf backup.tar
      │     └─ name of the archive you want to look inside
      └─ -t list, -f (always do f)

Paths — Absolute vs Relative

Absolute path: starts with / and points from the root of the filesystem. Works from anywhere. Example: /etc/nginx/nginx.conf.

Relative path: does not start with /. It's interpreted from your current directory (what pwd shows). Examples: docs/report.txt, student/images, ./run.sh.

Home shortcut: ~ expands to your home directory (e.g., /home/student), so ~/Downloads is an absolute location after expansion.

Absolute vs Relative in practice
cd /var/log
      └─ absolute: starts at / (root), valid from anywhere
Go to a subfolder (relative)
cd student/images
      └─ relative: “student/images” from where you are now
Home shortcut
cd ~/Downloads
      └─ “~” expands to /home/student (your home)
